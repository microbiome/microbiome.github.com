Fetch measurements associated with one or more extractions from a phyloarray database

Usage

fetch.measurements(con, extrList, which.data = c("raw", "rawBGcor", "spatnorm", "qnorm"),
  transformation = "none")

Arguments

con
MySQL connection
extrList
An extractionList object obtained from fetch.extractions
which.data
One or more of raw, rawBGcor, spatnorm and qnorm
transformation
One of none, avg, log or avglog

Value

A dataframe with the selected measurements

Description

The function fetches records from the featureMeasurements table, including some fields from the array, hybridisation and featureExtraction tables. The which.data argument indicates which data to fetch from the database, where raw will fetch the raw data (FGsignal), rawBGcor will fetch the raw background corrected data (FGsignal-BGsignal), spatnorm will fetch the spatially normalized data and qnorm will fetch the sample-quantile normalized data. The transformation argument indicates which transformation to apply to the data indicated by the which.data argument. Only one type of transformation is allowed, where "none" will not apply a transformation, "avg" will take the values averaged over oligo's and featureExtractions including standard deviation, "log" will take the 10-base logarithm of te data and "avglog" will return the oligo and featureExtraction averaged values of the logarithm of the data, including standard deviation.
